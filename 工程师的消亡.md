# 工程师的消亡

---

## LeetCode 已经不重要了

过去十几年，工程师的入场券叫做算法题。

你背了两百道 LeetCode，能手写红黑树，能在白板上默写 DP 转移方程，你就能进大厂，拿高薪，赢得游戏。

这套逻辑背后有一个假设：**写代码的能力是稀缺的，能写对算法的人是少数**。

AI 把这个假设摧毁了。

现在任何人打开 Claude 或 Copilot，把题目粘进去，三秒钟得到一个比 99% 面试者都写得好的答案。动态规划、图论、线段树——AI 全会，而且不会紧张，不会忘，不会手抖。

考算法，考的是你能不能做 AI 已经做得更好的事。这件事失去了意义。

---

## 新的考核标准：你够不够耐操

那考什么？

考你能不能在没有人告诉你该做什么的情况下，独立把一个系统从零到生产跑起来。

这意味着你需要掌握的，不再是一个点，而是整条链：

### 前端
- HTML / CSS / JavaScript 基础
- React / Vue / Next.js / Nuxt
- TypeScript
- 状态管理（Zustand、Pinia、Redux）
- 构建工具（Vite、Webpack、Turbopack）
- 测试（Vitest、Playwright、Cypress）

### 后端
- Node.js / Python / Go / Java / Rust（至少两门）
- REST API 设计、OpenAPI 规范
- GraphQL
- 数据库：PostgreSQL、MySQL、Redis、MongoDB
- ORM / Query Builder（Prisma、SQLAlchemy、GORM）
- 消息队列（Kafka、RabbitMQ、NATS）
- 缓存策略、限流、熔断

### 高并发与系统设计
- 分布式系统基础（CAP、BASE、一致性模型）
- 微服务架构、服务网格（Istio）
- 负载均衡、反向代理（Nginx、Envoy）
- 分布式事务、幂等设计
- 可观测性（Prometheus、Grafana、OpenTelemetry、Jaeger）

### 部署与基础设施
- Docker、容器化最佳实践
- Kubernetes（Pod、Deployment、Service、Ingress、HPA、PVC）
- Helm Charts
- Terraform / Pulumi（Infrastructure as Code）
- 云平台：AWS / GCP / Azure（至少一个）
- Serverless（Lambda、Cloud Run、Cloudflare Workers）

### CI/CD 与工程实践
- Git 工作流：GitHub Flow、Trunk-Based Development
- GitHub Actions / GitLab CI / CircleCI
- 代码审查流程、Branch Protection Rules
- 语义化版本（Semantic Versioning）
- 自动化测试流水线（单元 → 集成 → E2E）
- 蓝绿部署、金丝雀发布、Feature Flags
- DORA 指标（部署频率、变更失败率、恢复时间）

### 安全
- OWASP Top 10
- OAuth2 / OIDC / JWT
- Secrets 管理（Vault、AWS Secrets Manager）
- SBOM、依赖扫描（Snyk、Dependabot）

---

这一张清单，过去需要一个五到十人的团队才能覆盖。

现在，你一个人，加上 AI，理论上都要能搞定。

这不是"全栈工程师"，这是**把整个工程团队的职责压到一个人身上**。

> 这简直就是架在火上烤。

---

## 信息差消失了，你的护城河也消失了

以前的信息差是真实存在的。

- 知道怎么调 JVM GC 参数的人，比不知道的人值钱
- 看过 Nginx 源码的人，比没看过的人更能解决线上问题
- 去过大厂、见过大流量的人，比没见过的人经验更值钱

AI 把这些信息差全部夷平了。

你花三年积累的 Kubernetes 调优经验，我花三天问 AI，得到的答案质量不见得比你差。你踩过的坑，我不需要亲自踩，AI 帮我总结，附带三种解决方案和每种方案的 trade-off。

于是所有人的简历开始趋同：

> 精通 Kubernetes、熟悉分布式系统、有高并发经验、掌握 CI/CD 全流程……

**你的优势在哪里？**

说不出来。因为这些东西，人人都能写，也人人都能用 AI 快速补课。

## 考证书的时代来了，护城河越来越虚

绝望之下，很多工程师开始做一件事：**考证书**。

AWS Certified Solutions Architect、CKA（Kubernetes 管理员认证）、Google Cloud Professional、各种 Security 认证……

逻辑是这样的：如果所有人的技术能力都趋同，那我有一张官方认证的纸，我是不是就有优势？

也许短期内有用。但这条路的终点很清楚：

- 证书越来越多 → 越来越不值钱
- 考证的人越来越多 → 门槛抬高但区分度下降
- AI 辅助备考越来越强 → 考试本身失去筛选意义

你在忙于制造**一道根本不存在的护城河**。

---

## 真正的竞争变成了什么

把那张技术清单放在一边。真正在筛选人的，已经不是技术本身，而是：

**你能不能在极度模糊的情况下做出正确的判断？**

- 这个需求该不该做？
- 这个架构选择的 trade-off 是什么？
- 这个 bug 背后的根因在哪里？
- 这个系统三个月后会在哪里炸？

这些问题，AI 可以给你选项，但不能替你做决定。因为做决定需要你对上下文、对业务、对团队、对用户有真实的理解——而这些是无法从 prompt 里传进去的。

还有一个更残酷的维度：**你能不能持续高压工作而不崩溃？**

工具让每个人的上限都提高了，但也让每个人的下限变得更容易暴露。稳定性、抗压能力、在混乱中保持判断力——这些"软性"能力，正在成为真正稀缺的东西。

---

## 工程师没有消亡，但那个职业的定义变了

以前的工程师：用技术技能换薪水。

现在的工程师：用判断力 + 系统思维 + 抗压能力，驱动 AI 产出结果，然后为结果负责。

这不是降级，是另一种能力的稀缺。

只是很多人还没意识到，他们正在用全部精力磨一把正在变钝的刀。
