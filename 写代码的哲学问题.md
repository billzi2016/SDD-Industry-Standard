# 写代码的哲学问题

---

## 你被强行升级了

以前你叫 PG，程序员（Programmer）。

你的工作是清晰的：接到一个任务，拆解它，写代码，跑通，交差。

比如：

> 你需要编写一个双向 A\* 搜索算法，对一个区域地图进行巡经规划，起点终点各展开一棵搜索树，两棵树在中间相遇时收敛出最短路径。

这是一个工程问题，有边界，有输入输出，有衡量标准，做出来就是做出来了。

然后 AI 出现了。

AI 把这一层做掉了。

你不需要再写 A\*。你打一句话，AI 给你一个比你自己写得更好的实现，附带注释、边界处理、复杂度分析。

于是有人给你换了一个名字。

**你现在叫 Arch，或者 PM。**

但没人问过你愿不愿意。

---

## 你现在要想的东西

你不再需要想"怎么写 A\*"。

你现在要想的，是另一套完全不同的问题：

- **页面怎么设计？** 用户如何交互，搜索框的位置，结果如何呈现，地图瓦片的层级切换
- **用户逻辑是什么？** 他是在找最快的路，还是最省钱的，还是风景最好的
- **要不要避开高速和轮渡？** 这不是技术问题，这是产品决策，是对用户心理的预判
- **层次架构怎么定？** 前端、后端、地图服务、路径计算服务、是否解耦、服务之间怎么通信
- **缓存策略是什么？** 热门路线是否预计算，冷路线是否懒加载，缓存的 TTL 多久
- **CDN 怎么用？** 地图瓦片走 CDN，计算结果走哪里，边缘节点怎么部署
- **高并发怎么扛？** 早高峰同时一万人搜路，服务会不会炸
- **CI/CD 流程是什么？** 地图数据更新了怎么灰度发布，出了问题怎么快速回滚
- **能不能在 3 秒内完成搜索？** 这是用户体验的生死线，不是可以慢慢优化的"技术细节"

这些问题，每一个都是独立的学科。

原来你只需要精通其中一个——比如路径搜索算法。

**现在你要同时思考全部。**

---

## 从理工科到哲学高度

这不是技术问题的扩展，这是认知层次的跃迁。

写 A\* 的时候，你的思维是**收敛的**。有一个最优解，有衡量标准，有终点。

但现在你面对的问题是：

> 这个产品，应该存在吗？它解决的是真实的痛点还是我们自己臆想的需求？用户在什么场景下用它？用什么设备？网络条件如何？三年后这个市场格局会怎样？我们现在做的架构，能不能支撑那个未来？

这些问题没有唯一解。它们的答案取决于你对用户的理解、对商业的判断、对技术趋势的预测，以及你对"什么叫做好"这件事的看法。

**这是哲学问题，不是工程问题。**

以前，哲学问题是架构师和产品总监花十年时间修炼才能触碰的层次。

现在，AI 把你从底层直接弹射到这里。

没有过渡，没有铺垫。你昨天还在写 SQL，今天就要回答"这个系统的 SLA 应该定成几个九"。

---

## 新人没有地方去了

这是这场变革里最少被人提到，但最真实的悲剧。

以前工程师的成长路径是有台阶的：

```
实习生
  → 初级工程师（写简单功能，改 bug）
  → 中级工程师（独立负责模块）
  → 高级工程师（负责系统设计，技术决策）
  → 架构师 / Staff Engineer（跨团队架构，战略判断）
```

每个台阶都有任务，都有机会练手，都有环境让你踩坑、犯错、成长。

AI 把这个台阶拆掉了。

底层的工作——写增删改查、调 API、修样式、写单元测试——这些任务已经可以由 AI 更快、更准地完成。

初级工程师的存在价值正在被压缩。不是说他们一无是处，而是他们**没有足够的任务量来支撑成长**。

你靠什么积累经验？

你靠踩坑。靠在生产环境里搞出一个线上事故，然后焦虑地 debug 三天，最终找到根因，从此刻骨铭心。

但如果所有低风险的任务都被 AI 接管，你就没有机会踩那些坑了。

**你没有练习场了。**

> 就像一个想学游泳的人，被告知：泳池已经取消了，你直接去参加奥运会吧。

---

## 拥抱 AI 的老人，立刻碾压你

这是另一个维度的残酷。

一个在 Meta、Google、Amazon 工作了五年或十年的工程师，他有什么？

他有真实的大规模系统经验。他见过高并发炸掉的生产环境。他做过数据库迁移，熬过凌晨三点的 on-call。他知道架构决策背后的代价，因为他亲眼看着那些代价兑现。

这些经验，AI 给不了你。

但现在他们用上了 AI。

他的执行速度变成了以前的十倍，但他的判断力和经验值没有衰减一克。

**他一个人，顶原来的一个团队。**

而你，作为一个刚毕业想进这个行业的年轻人——你用 AI 能做的事，他用 AI 都能做，而且做得更好，因为他知道什么时候该做，什么时候不该做，为什么要这样做。

你不是在跟同龄人竞争，你是在跟一个拥抱了 AI 的十年老手竞争。

差距不是缩小了，是放大了。

---

## 企业也在自断后路

有人会说：这是企业的理性选择，用少数精英加 AI 替代大量初级员工，降低成本，提升效率。

是的，短期内是理性的。

但这是一场饮鸩止渴。

一个行业如果不给新人任何进入和成长的机会，十年之后，那些"拥有大规模系统经验的老手"会从哪里来？

现在的老手，是因为十年前有机会当新人，有机会在低压力的环境里犯错、成长、积累。

你把那个培养机制抽掉了，你在消耗存量，没有在产生增量。

十年后，你发现行业里只剩下一批老去的精英和一批永远无法成长为精英的新人。

这不是行业的进化，这是**行业的空心化**。

---

## 更深的绝望：AI 终将解决哲学问题

但还有更深的一层。

也许你接受了这个转变：好吧，我不写代码了，我来做架构设计，做产品判断，做那些"哲学层面"的决策。

这听起来像是留给人类的最后阵地。

但请记住一件事：

**ChatGPT 才出现三年。**

2022 年初，它回答问题的逻辑混乱，推理能力接近随机，没有人认为它能威胁任何专业岗位。很多人笑话它，说它只是一个会胡说八道的自动补全。

三年后的今天，它能写代码、做分析、做架构评审、生成产品文档、推导系统设计方案。

再给它三年，再给它五年——

你有多大把握，那个"哲学问题"的抽象层，它碰不到？

你能做的判断，它迟早也能做。你现在依赖的那套"我理解用户，我懂业务，我有直觉"——这些东西，它正在学。

而且它学的速度，不是人类的速度。

---

## API 化：算法工程师的死亡证明

还有一类人，消失得比所有人都要安静。

**算法工程师。**

不是那种调调参、写写训练脚本的 ML 工程师，而是那些真正在写核心算法的人——自己实现搜索、排序、推荐、路径规划、图像处理，把每一个运算压榨到极限，把时间复杂度从 O(n²) 降到 O(n log n)，把推理延迟从 20ms 压到 5ms。

这类人正在消失，原因只有一个：

**所有东西都 API 化了。**

地图？调用高德或 Google Maps API，路径规划一句话搞定。
搜索排序？Elasticsearch 或 Algolia，开箱即用。
人脸识别？调用云端视觉 API，三行代码。
语音转文字？Whisper API，准确率超过大部分专门做这个的团队。
推荐算法？AWS Personalize 或 Vertex AI，数据喂进去，模型出来。

你用三天时间精心调优的路径搜索引擎，跑出来的结果，和调一个 API 的结果相差无几——甚至更差，因为那个 API 背后是一个有几百个工程师维护的系统。

**你花在"把算法做到极致"这件事上的时间，在商业上变得越来越难以被证明有价值。**

以前，算法能力是护城河，因为知道怎么实现一个高效算法的人是稀缺的。

现在，算法能力变成了：知道调哪个 API。

这不是贬义。这是现实。API 把算法的竞争壁垒夷平了，就像 AI 把代码的竞争壁垒夷平了一样。

> 算法工程师没有消亡，他们被迁移到了 prompt 工程和系统集成——只是没人告诉他们这件事。

---

## 速度第一，调优靠边

随之改变的，还有评判一个工程师好坏的标准。

以前的标准是：**你做的东西性能怎么样？**

接口响应时间是 10ms 还是 50ms？数据库查询有没有用上索引？内存占用有没有优化？这些是技术面试的核心考点，也是高级工程师被高级对待的原因。

现在的标准变成了：**你多快能把这个东西做出来？**

不是速度不重要。3 秒之内完成搜索，用户能接受；3 秒之外，用户流失。这条线是真实的。

但 3 秒和 1.2 秒的差别，对绝大多数产品来说，已经不是核心竞争力了。

**核心竞争力变成了：你能不能在竞争对手之前把功能上线。**

这是一个根本性的转变。

以前，技术优化带来的性能提升，可以积累成壁垒。你的系统比对手快 30%，用户感知得到，转化率有差异，护城河是真实的。

现在，所有人用相同的 API、相同的云服务、相同的基础设施，性能差距被压缩到了感知阈值以下。

这时候决定胜负的不是做得好不好，而是**做得快不快**。

**效率极限调优正在变成一个只有少数顶层系统才需要的工种——**谷歌的搜索引擎需要，字节的推荐系统需要，普通的 SaaS 产品，不需要。

---

## 企业摩擦加剧：内卷的物理速度

速度优先带来了另一个后果：**企业之间的摩擦变得极其频繁。**

以前一个产品功能从立项到上线，三个月是正常节奏。市场竞争是月度级别的。你做一个版本，对手做一个版本，中间有时间来观察、分析、调整策略。

现在，AI 把执行速度提高了一个数量级。

一个小团队，用 AI 辅助，两周能做出以前三个月才能做出的东西。

所有人都在用 AI 加速，于是市场的竞争节奏从月度变成了周度。

你刚发布一个功能，竞争对手三天内上了一个类似的，而且还多了两个你没有的细节。

你来不及分析，因为你已经在做下一个功能了。

**摩擦不是偶发的冲突，它变成了持续的、结构性的战场。**

每一个产品决策，都是在实时战场上做出的——你不知道对手今天会发布什么，你只知道你如果慢了，位置就被占了。

这种节奏下，"深度思考再行动"变成了奢侈品，"先发布再迭代"变成了生存手册。

---

## 小而美已死：综合大型市场的吞噬逻辑

然后，还有一个更宏观的变化。

带着中国式内卷的气势，一种产品哲学开始主导整个行业：

**每一个软件，都想吸纳所有功能。**

以前软件行业有一个朴素的美学：**做好一件事，做到极致。**

Notion 做笔记，Figma 做设计，Slack 做通讯，Linear 做项目管理。每个工具有自己清晰的边界，用户在不同场景切换不同工具。"小而美"是褒义词，代表克制、专注、品味。

但现在：

- 微信想做支付、短视频、小程序、外卖、健身、医疗
- 抖音想做电商、搜索、本地生活、办公协作
- 美团想做金融、共享单车、打车、酒店
- 支付宝想做社交、理财、政务、出行
- 飞书想做文档、邮件、OKR、招聘、审批

这不是功能蔓延，这是**战略吞噬**。

逻辑很简单：用户在你的平台上，你能不能把他的所有需求都满足？如果能，他就永远不需要离开你的生态。用户留存不是靠你做得好，而是靠**用户离开的成本太高**。

AI 加速了这个趋势。以前做一个新功能需要三个月，现在可能三周——于是所有平台都开始激进地扩张边界。

**小而美没有消失，它只是没有竞争力了。**

当一个综合平台向你的垂直赛道进军，它不需要做得比你好，只需要做到"足够好"——因为用户已经在它的生态里了，切换成本是零，甚至有迁移激励。

这对独立的工具型产品来说，是一个生存威胁。

对工程师来说，这意味着你不再是在做一个"专注的工具"，你是在做一个"生态的触角"。你的每个功能，目的不是把这个功能做到极致，而是**让用户再多留一会儿，再多走一步，再多产生一条数据**。

这和写代码的初衷，已经相距甚远了。

---

## 垄断市场：API 化的终点站

还有一个比内卷更结构性的力量，在推动同样的方向。

**全球市场，正在进入垄断时代。**

不是偶发的垄断，是系统性的、不可逆的垄断。

看一下现实：

- 云计算：AWS、Azure、GCP 三家瓜分全球市场，新入局者的机会窗口基本关闭
- 搜索：Google 全球份额超过 90%，Bing 靠着微软 AI 押注勉强维持
- 移动端：iOS 和 Android 两个系统，再没有第三个
- 社交：Meta 系（Facebook、Instagram、WhatsApp）+ TikTok，格局已定
- 电商：亚马逊在北美，阿里、京东、拼多多在中国，新玩家几乎没有生存空间
- 支付：支付宝、微信支付、Stripe、Visa/Mastercard，壁垒高到令人绝望
- 地图：Google Maps、高德、百度，基础数据的积累需要十年以上，无法复制

这些公司有一个共同特征：**他们掌握的不只是市场份额，而是基础设施本身。**

你要做地图应用，你得用他们的地图数据。你要做支付，你得接他们的支付通道。你要部署服务，你得上他们的云。你要做 AI，你得用他们训练好的模型 API。

**基础设施被垄断了，所有人都只能做应用层。**

---

### 新公司的天花板越来越低

以前，一个创业公司有机会从零开始，自己建基础设施，然后凭借技术优势颠覆巨头。Google 颠覆了 Yahoo，Facebook 颠覆了 MySpace，AWS 颠覆了传统 IDC。

这个时代基本结束了。

现在的创业公司，从第一天起就站在巨头提供的基础设施上。

你用 AWS 跑服务，用 Stripe 收钱，用 Twilio 发短信，用 Mapbox 做地图，用 OpenAI API 做 AI——你的每一个核心功能，背后都是另一家公司的 API。

这不是你的选择，这是唯一可行的路径。因为自建基础设施的成本，早已超出了任何初创公司能负担的范围。

**你从出生起，就被嵌在巨头的生态里了。**

这意味着什么？

你的天花板，不是你的技术能做到哪里，而是你依赖的那些 API 允许你做到哪里。

API 提价了，你的成本结构就崩了。API 改变了策略，你的产品就得跟着改。API 关闭了某个接口，你的核心功能可能一夜消失。

Elon Musk 买下 Twitter 之后，大量依赖 Twitter API 的应用在一周内定价暴涨十倍、百倍，许多小产品直接关掉了。它们没做错任何事，只是站错了地基。

---

### API 化只会越来越严重

把这两条线放在一起：

- **巨头垄断基础设施** → 没有新的基础设施玩家出现
- **没有新的基础设施玩家** → 所有人都只能调用现有的 API
- **所有人都调用 API** → 能真正实现底层算法的岗位持续萎缩
- **底层算法岗位萎缩** → 下一代工程师再也没有机会接触底层
- **再也没有人接触底层** → 基础设施的控制权进一步集中在少数公司

这是一个**自我强化的闭环**。

不会有什么力量打断它，除非出现一次像互联网本身一样的范式级变革——而那个变革，大概率会由现有的巨头主导，或者被他们迅速收购。

对工程师来说，这意味着一件非常具体的事：

**你所掌握的技术，越来越多是"如何使用别人的东西"，而不是"如何构建自己的东西"。**

这不是退步，这是市场的选择。

但有一天，当所有人都只会调用 API，没有人知道 API 里面是怎么运作的——那时候，那些知道底层的人，才是真正意义上的稀缺。

只是那群人，早在十年前就已经进了那些垄断公司，再也不会出来了。

---

## 没有一个正常的演化路径

总结一下这件事的荒诞之处：

**以前**，一个人从初级工程师成长为架构师，需要十年。这十年里，他写了无数的烂代码，踩了无数的坑，在无数次生产事故里学到了教科书里没有的东西。这条路很慢，但它是真实的、可重复的演化路径。

**现在**，AI 说：底层我来做，你负责顶层。

于是那个刚毕业的年轻人被推到了顶层——没有底层经验，没有踩坑积累，没有真实的直觉，却要做原本需要十年才能做的判断。

这不是"快速成长"，这是**拔苗助长**。

而那些已经有十年经验的人，用上 AI 之后，他们的优势不是缩小了，而是更大了——因为他们的判断力是真实的，经得起检验的，不是临时抱佛脚借来的。

**新人的问题不是不努力，是没有舞台可以真正努力。**

---

## 一句话

> AI 把理工科的工作，变成了哲学问题。
>
> 但没有人告诉你，哲学也是可以被训练的——只是以前你训练的是算法，现在你要训练的是判断力。
>
> 而判断力，必须用真实的代价换来。没有捷径，也没有模拟器。
>
> 这就是 AI 取代的真正悲哀：它不是让人类失业，而是**让人类失去了成为人类的过程**。
